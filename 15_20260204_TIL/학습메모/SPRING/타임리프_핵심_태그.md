### 타임리프 핵심태그 정리

## 1-1. Thymleaf 용어풀이
 서버사이드 템플릿 엔진 : 서버에서 HTML을 동적으로 생성해서 사용자에게 전달하는 기술

## 1️⃣ 텍스트 및 기본 출력 태그(가장 기본이 되는 출력 관련 태그)
1. **th:text** : 태그 안의 텍스트를 데이터로 바꿔준다.
    예시: <p th:text="${variable}">기본 텍스트</p>
    화면출력: "variable" 변수의 값 
    활용빈도: ★★★★★
2. **th:utext** : th:text와 동일하지만, HTML 태그가 포함된 문자열을 그대로 출력.
    예시: <div th:utext="${htmlContent}">기본 HTML</div>
    화면출력: "htmlContent" 변수의 값 (HTML 태그가 적용된 상태)
    활용빈도: ★★★☆☆
3. **th:value** : <input> 태그 등의 value 속성값을 설정함
    예시: <input type="text" th:value="${inputValue}" />
    화면출력: inputValue 변수의 값이 input 필드에 기본값으로 설정됨
    활용빈도: ★★★★☆

## 2️⃣ 조건문 및 반복문(핵심 로직)
1. **th:each** : 컬렉션(리스트, 배열 등)을 반복하여 출력할때 사용
    예시: <tr th:each="item : ${itemList}"> ... </tr>
    화면출력: itemList의 각 항목에 대해 tr 태그가 반복 생성됨
    활용빈도: ★★★★★
2. **th:if** : 조건이 참(true)일 때만 해당 태그를 보여줌.
    예시: <div th:if="${isVisible}">보여질 내용</div>
    화면출력: isVisible이 true일 때만 내용이 보임
    활용빈도: ★★★★★
3. **th:unless** : 조건이 거짓(false)일 때만 해당 태그를 보여줌.
    예시 : <div th:unless="${isHidden}">보여질 내용</div>
    화면출력 : isHidden이 false일 때만 내용이 보임
    활용빈도: ★★★★☆
4. **th:switch** & **th:case** : 여러 조건 중 하나를 선택하여 출력할 때 사용
    예시:
    <div th:switch="${status}">
        <p th:case="'ACTIVE'">활성 상태</p>
        <p th:case="'INACTIVE'">비활성 상태</p>
        <p th:case="*">알 수 없는 상태</p>
    </div>
    화면출력: status 변수의 값에 따라 해당 case의 내용이 출력됨
    활용빈도: ★★★☆☆

## 3️⃣ 속성 변경 및 링크(주로 HTML 속성을 동적으로 변경할 때 사용)
1. **th:href** : <a> 태그의 링크 주소를 동적으로 생성.
    예시: <a th:href="@{/board/detail(id=${board.id})}">상세보기</a>
    화면출력: /board/detail?id=board.id 값으로 링크 생성
    활용빈도: ★★★★★
2. **th:src** : <img>, <script> 등의 경로를 동적으로 설정.
    예시: <img th:src="@{/images/${imageFileName}}"/>
    화면출력: /images/imageFileName 값으로 이미지 경로 설정
    활용빈도: ★★★★☆
3. **th:action** : <form> 태그의 제출 경로를 동적으로 설정.
    예시: <form th:action="@{/board/save}" method="post"> ... </form>
    화면출력: 폼 제출 시 /board/save 경로로 전송
    활용빈도: ★★★★★
4. **th:classappend** : 기존 클래스에 조건부로 클래스 추가
    예시: <div th:classappend="${isActive} ? 'active' : ''">내용</div>
    화면출력: isActive가 true일 때 'active' 클래스가 추가됨
    활용빈도: ★★★☆☆

## 4️⃣ 레이아웃 관련 태그(여러 페이지에서 공통으로 쓰는 상단, 하단 관리)
1. **th:fragment** : 공통으로 사용할 코드 조각에 이름을 붙여 정의로
    조각의 정의라고 생각하면 됨
    예시 : <div th:fragment="header"> ... </div>
    활용빈도: ★★★★☆
2. **th:replace** : 현재 태그를 지정한 코드 조각으로 대체(교체)
    why? 실무에서 페이지가 100개 가 넘게 된다면 한 번만 고치면 다 바뀌기 때문이다.
    예시 : <div th:replace="~{fragments/header :: header}"></div>
    화면출력: fragments/header.html 파일의 header 조각으로 대체
    활용빈도: ★★★★★
3. **th:include** : 현재 태그 안에 지정한 코드 조각을 포함(삽입)
    예시 : <div th:include="~{fragments/footer :: footer}"></div>
    화면출력: fragments/footer.html 파일의 footer 조각이 삽입됨
    활용빈도: ★★★☆☆
