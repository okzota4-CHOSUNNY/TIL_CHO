## 3계층 아키텍처 (3-Tier Architecture)

1. **@Controller**: 사용자의 요청을 받고, 처리 결과를 반환하는 역할.
2. **@Service**: 비즈니스 로직을 처리하는 핵심 계층.
3. **@Repository / @Mapper**: 데이터베이스와 직접 소통하여 데이터를 처리.

---

## 프로젝트 폴더별 역할 (Security 기반)

### config (설정)
- **SecurityConfig.java**: 인증/인가 정책, 로그인/로그아웃 설정 등 보안의 중심.

### controller (제어)
- **AdminController**: 관리자 전용 기능 매핑.
- **BoardController**: 게시물 CRUD 요청 처리.
- **HomeController**: 메인, 로그인, 회원가입 등 공통 페이지.

### DTO (Data Transfer Object)
- **Users, UserAuth, Board**: DB 테이블과 매핑되거나 계층 간 데이터를 실어 나르는 바구니.
- **CustomUser**: 시큐리티 전용 유저 정보 객체.

### service (비즈니스 로직)
- **UserService / BoardService**: 실제 업무 흐름 처리.

### mapper (DB 연동)
- **UserMapper / BoardMapper**: MyBatis SQL 매핑 인터페이스.
