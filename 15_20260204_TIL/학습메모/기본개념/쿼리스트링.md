query 뜻: 질문, 조회, 탐색
DB에서 데이터를 조회하거나 조작하기 위해 사용하는 명령어

1. 기본구조:
https://example.com/path?key1=value1&key2=value2&key3=value3
                        └──────────────┬──────────────┘
                                   쿼리 스트링
1-1. 시작기호(?=시작):
 -? : 쿼리 문자열의 시작을 나타냄(URL에서 ? 뒤에 오는 부분을 쿼리문자열 이라함, 한번만 등장)
1-2. 파라미터 형식(key=value):
 -key=value형태: 등호(=)를 기준으로 key와 value로 나뉨(key: 매개변수 일름, value: 값)
1-3. 여러파라미터 연결(name1=value1&name2=value2):
 -& : 앰퍼샌드(&)기호로 여러개의 파라미터 연결 
 예시: ?name=조성진&age=25&city=seoul(3개의 파라미터가 연결됨)

2. 같은 키 여러번 사용
?color=red&color=blue&color=green 같은 키 여러번 사용 가능

3. URL인코딩
 -특수문자는 인코딩해야 함.
 예: 원본 ?name=홍길동&msg=안녕하세요!
    인코딩 ?name=%ED%99%8D%EA%B8%B8%EB%8F%99&msg=%EC%95%88%EB%85%95%ED%95%98%EC%84%B8%EC%9A%94!

4. 대소문자는 구분된다
 ?Name=value # Name
 ?name=value # name 
 위아래는 다른 파라미터로 인식되어진다.
