## HTML(HyperText Markup Language)
- HyperText: 클릭하면 다른 곳으로 이동할 수 있는 '링크' 기능을 가진 텍스트
- Markup Language: 데이터를 설명하는 태그를 추가하는 언어

## 렌더링(Rendering)
- 뜻: 텍스트로 된 코드(HTML, CSS, JS 등)를 브라우저가 해석하여 시각적으로 보여주는 과정

## SSR(Server-Side Rendering)
- 서버가 주도권을 가짐: 서버에서 페이지를 완전히 렌더링하여 완성된 HTML을 클라이언트에게 제공
- 특징: 빠른 초기 로딩, SEO 유리, 서버 부하 증가

## CSR(Client-Side Rendering)
- 클라이언트(브라우저)가 주도권을 가짐: 서버는 데이터(JSON)만 보내고 브라우저가 JS로 페이지를 동적 생성
- 특징: 초기 로딩 느릴 수 있음, 부드러운 화면 전환, 서버 부하 감소

## JSP(JavaServer Pages)
- 자바 코드를 HTML 안에 삽입하여 동적인 웹페이지를 생성하는 서버 사이드 기술
- 사용자가 요청할 때마다 그때그때 다른 내용을 보여줄 수 있음
