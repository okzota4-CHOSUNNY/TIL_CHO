```java
1단계: DTO (Data Transfer Object)
파일 위치: src/main/java/.../dto/
이유: SQL로 만든 테이블의 데이터를 자바 객체로 담아야 합니다.
테이블의 컬럼 이름과 타입을 자바 변수로 매핑하는 가장 기초적인 작업입니다.
예시: TutorDocument.java, Users.java 등

2단계: Mapper Interface
파일 위치: src/main/java/.../mapper/
이유: DB에 어떤 쿼리를 날릴지 함수 이름(메소드)으로 정의하는 곳입니다.
"데이터를 가져와라", "수정해라" 같은 명령의 목록이라고 보시면 됩니다.

3단계: Mapper XML (MyBatis SQL)
파일 위치: src/main/resources/.../mapper/
이유: 2단계에서 만든 인터페이스와 연결되어 실제 실행될 SQL 쿼리를 작성하는 곳입니다.
DB 테이블과 1단계 DTO를 연결해주는 핵심 다리 역할을 합니다.

4단계: Service Interface & Implementation
파일 위치: src/main/java/.../service/
이유: 단순한 DB 연동을 넘어 "비즈니스 로직"을 처리하는 곳입니다.
예를 들어 "승인 버튼을 누르면 DB 상태를 바꾸고, 동시에 로그도 남겨라" 같은 구체적인 업무 규칙을 작성합니다.

5단계: Controller
파일 위치: src/main/java/.../controller/ (또는 api/)
이유: 사용자의 도구(브라우저)에서 온 요청을 가장 먼저 받는 관문입니다.
"이 주소(/admin)로 들어오면 이 서비스를 실행해!"라고 지시하고 데이터를 화면으로 전달합니다.

6단계: View (Thymeleaf/HTML)
파일 위치: src/main/resources/templates/
이유: 마지막으로 사용자에게 보여줄 화면입니다.
컨트롤러가 넘겨준 데이터를 HTML에 예쁘게 뿌려주는 단계입니다.
