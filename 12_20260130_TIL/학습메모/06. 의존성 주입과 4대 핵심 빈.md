## 의존성주입(DI; Dependency Injection)
Dependency 뜻: 의존성, Injection 뜻: 주입

## DI는 컴포넌트 간의 의존성을 외부에서 주입해주는 설계 패턴입니다.
컴포넌트란? 소프트웨어를 구성하는 독립적인 부품(모듈)을 의미합니다.
    ex: 클래스, 함수, 라이브러리 등
    모듈이란? 독립적으로 기능을 수행하는 소프트웨어의 구성 요소

## 스프링 DI
- 스프링 프레임워크에서는 DI를 쉽게 구현할 수 있는 기능을 제공.

## IOC 컨테이너(Inversion of Control Container)
- Inversion뜻: 역전, Control 뜻: 제어
- 개발자는 컴포넌트 간의 의존성을 직접 관리하지 않고, IOC 컨테이너가 이를 대신 관리
- 스프링 IOC 컨테이너는 애플리케이션의 컴포넌트들을 생성, 구성, 관리하는 역할을 담당
- 스프링 컨테이너는 애플리케이션이 시작될 때 필요한 컴포넌트들을 생성하고, 이들 간의 의존성을 주입하여 연결

## 빈(Bean) 정의
- 개발자가 스프링 IoC 컨테이너에 의해 관리되는 객체를 의미
- 스프링 빈은 일반적으로 
@Component, @Service, @Repository, @Controller 등의 어노테이션을 사용하여 정의

### @Component
- 스프링 컨테이너는 애플리케이션이 시작될 때 이들 빈을 생성하고, 필요한 의존성을 주입하여 관리
- 컴포넌트의 아주 쉬운 개념: 
    완성된 레고 성을 '웹사이트', '프로그램' 이라고 한다면
    열고 닫히는 성문, 성벽을 지키는 기사 등을 '컴포넌트' 라고 할 수 있다.
- 컴포넌트의 중요한 개념 3가지는
    1. 독립성: 각 컴포넌트는 독립적으로 기능을 수행할 수 있어야 한다.
    2. 역할: 각 컴포넌트는 특정한 역할이나 기능을 담당해야 한다.
    3. 재사용성: 컴포넌트는 여러 곳에서 재사용될 수 있어야 한다.
- 예시: Config(설정) 컴포넌트:
    - 독립성: 설정 정보를 다른 컴포넌트와 독립적으로 관리
    - 역할: 애플리케이션의 설정 정보를 관리
    - 재사용성: 여러 컴포넌트에서 동일한 설정 정보를 참조 가능
- Constant(상수) 컴포넌트:
    - 독립성: 상수 값들을 다른 컴포넌트와 독립적으로 관리
    - 역할: 애플리케이션에서 사용하는 상수 값을 정의
    - 재사용성: 여러 컴포넌트에서 동일한 상수 값을 참조 가능

### @Service
- 비즈니스 로직 컴포넌트를 정의할 때 사용
    아주 쉬운 표현으로 우리 프로그램만의 특별한 규칙(Rule)을 처리 하는 컴포넌트
- 예시: 배달앱 서비스
    - 규칙1: 주문 금액이 2만원 이상일 때 무료배송
    - 규칙2: 첫 주문 고객에게는 30% 할인
- @Service의 중요 개념 4가지
    1. 비즈니스 로직의 응집: 컨트롤러는 길안내, 리포지토리는 창고보관, **어떻게 처리할 
    것인가는 서비스가 담당**
    2. 트랜잭션(Transaction)의 경계: @Service 매서드 하나는 하나의 큰 **작업단위**가 된다.
    결제를 예를 들면 **재고감소+결제 정보 생성+포인트 적립 등이 하나의 트랜잭션 단위가 된다.**
    3. 낮은 결합도와 높은 재활용성(부품의 '독립성'): 서비스는 특정 컨트롤러나 리포지토리에 강하게 의존하지 않고, 여러 컨트롤러나 리포지토드에서 재활용 가능
    4. 추상화 레이어(컨트롤러와 데이터 계층 사이의 중간다리): 서비스는 컨트롤러와 리포지토리 **사이**에서 비즈니스 로직을 처리하는 추상화 레이어 역할을 수행

### @Repository
- 컴포넌트 중에서 **데이터베이스(DB)** 창고와 직접 소통하는 부품
- 예시: UserRepository(사용자 저장 창고)
    - 역할: Oracle이나 MySQL 같은 DB에 접속해서 사용자 정보를 저장/조회/수정/삭제
- @Repository의 중요 개념
    1. 데이터 액세스 계층: 모든 DB 관련 코드를 모음. SQL쿼리를 직접 작성하거나, MyBatis, JPA 같은 ORM 프레임워크를 사용
    2. 예외 변환: DB마다 에러 메시지가 제각각(오라클, MySQL 등) 인데, 이를 스프링이 이해하기 쉬운 공통 언어로 바꿔준다.
    3. CRUD 기능: Create(생성), Read(조회), Update(수정), Delete(삭제) 같은 기본적인 DB 조작 기능을 제공
    4. 서비스 계층과의 분리: 서비스는 어떤 데이터를 어떻게 처리할지**만**(한가지일만) 고민하고, 실제 "DB에 어떻게 넣을지"는 리포지토리가 담당

### @Controller
- 컴포넌트 중에서 **사용자(클라이언트)** 요청을 받고 응답을 돌려주는 역할을 하는 부품
- 예시: BoardController(게시판 안내 데스크)
    - 역할: 웹 브라우저에서 /board/list 요청이 오면, 게시판 목록 화면을 보여줌
- @Controller의 중요 개념
    1. 요청 매핑(길 안내): @RequestMapping, @GetMapping, @PostMapping 등과 같은 어노테이션을 사용해 특정 URL 주소와 자바 메서드를 연결함 **(ex: /login 주소로 오면 login() 메서드 실행)**
    2. 파라미터 수집: 사용자가 입력한 아이디, 비밀번호, 게시글 내용등을 자바객체(DTO)로 변환해서 받아내는 역할
    3. 모델(Model) 전달: 서비스가 처리해온 데이터(게시글 목록 등)를 담아서 화면(HTML)으로 보내준다.
    4. 뷰(View) 선택: 어떤 화면(HTML, JSP 등)을 사용자에게 보여줄지 결정(return "board/list";)
