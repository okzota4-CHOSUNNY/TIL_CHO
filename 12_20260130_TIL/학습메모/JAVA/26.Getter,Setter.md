Getter와 Setter란?
1. Getter 메서드
- getter 메서드는 조회의 기능을 가진 메서드
- 반환값: 있음(안에있는걸 조회하자! 이니까 반환값이 있어야함)
- 사용예시: 게시글의 제목, 작성자, 내용 등 필드 값을 외부에서 조회할 때 사용
public String getTitle() {
// 해석: 모두에게 접근을 허용하는 String타입 의 getTitle 메서드
    return this.title;
// 해석: 이 클래스 내부에 있는 title 값을 반환(String 타입)
}

2. Setter 메서드
- setter 메서드는 수정의 기능을 가진 메서드
- 반환값: 없음(안에있는걸 바꾸자! 이니까 반환값이 필요없음)
- 사용예시: 게시글의 제목, 작성자, 내용 등 필드 값을 외부에서 수정할 때 사용
public void setTitle(String title) {
// 해석: 모두에게 접근을 허용하는 void타입(반환필요 없음) 의 setTitle 메서드
    this.title = title;
// 해석: 이 클래스 내부에 있는 title 값을 외부에서 전달받은 title 값으로 수정
// 반환할 필요가 없으니 return 구문이 없음
}
@Getter
전체적인 한줄 설명을 착한 깡패로 설명:
Getter: "어이~ 지갑좀 꺼내봐 얼마있냐?(돈은 안뺏고 보기만함)"
Setter: "야~ 돈이 어떻게 하나도없냐?! 돈좀 넣어둬!(돈을 넣음)"

+추가 설명
@Getter or @Setter를 클래스 이름 바로 위에 적으면, 해당 클래스의 모든 필드에 대해 자동으로 Getter/Setter 메서드를 생성해줌
@Getter or @Setter를 특정 변수 위에만 적으면, 해당 변수에 대해서만 Getter/Setter 메서드를 생성해줌
