```java
java의 4가지 접근제한자
1. public: 완전 개방(모든 클래스에서 접근 가능)
    public class Board; 
    - 접근범위: 누구나 접근 가능
    - 원론: 어떤 패키지, 어떤 클래스에서도 접근이 가능
    - 실제사용: 게시글 생성, 조회, 수정, 삭제 등 모든 기능에서 접근 가능해야 하는 게시글 클래스
2. protected: 부분 개방(같은 패키지 + 상속받은 자식에게만 접근 허용)
    protected String author;
    - 접근범위: 같은 패키지 + 상속받은 자식
    - 원론: 다른 패키지에 있더라도 '부모-자식' 관계라면 접근 가능
    - 실제사용: 공지사항 게시판은 특별히 작성자 정보를 상속받은 자식 클래스에서만 접근 가능해야 할 때
3. default: 생략된 상태(오직 같은 패키지 안에 있는 클래스끼리만 공유)
    String content;
    - 접근범위: 같은 패키지 안에서만 접근 가능
    - 원론: 오직 '같은 패키지' 안에 있는 클래스들끼리만 공유
    - 실제사용: 내부적으로만 사용되는 게시글 내용(content) 필드
4. private: 완전 폐쇄(오직 해당 클래스 내부에서만 접근 가능)
    private int viewCount;
    - 접근범위: 오직 해당 클래스 내부에서만 접근 가능
    - 원론: 객체지향의 핵심인 '캡슐화'를 실현하는 도구
    - 실제사용: 조회수(viewCount) 필드는 외부에서 직접 접근하지 못하도록 막고, 메서드를 통해서만 수정 가능
