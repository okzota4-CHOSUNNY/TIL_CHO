## Request
- HTTP Request(요청): 클라이언트가 서버로 보내는 요청 메시지, 웹 리소스에 대한 동작(GET, POST, PUT, DELETE 등)을 지정
- **중요개념**: 클라이언트 = 브라우저 가 서버에게 무언가를 요청
- GET 요청: 서버로부터 데이터를 조회할 때 사용
- POST 요청: 서버에 데이터를 생성하거나 제출할 때 사용
- PUT 요청: 서버의 기존 데이터를 수정할 때 사용

- 포스트맨과 같은 도구를 사용하는 이유
  1. 응답상태 코드(HTTP Status Code) 확인: 요청이 성공했는지, 실패했는지 상태 코드를 통해 쉽게 확인 가능(200, 404, 500 등)
  2. 응답 데이터(Response Body) 확인: 서버가 돌려주는 실제 데이터를 확인 가능(JSON, XML 등)
  3. 헤더(Header) 설정: 요청과 응답에 포함되는 메타데이터를 쉽게 설정하고 확인 가능
  4. 앤드포인트 및 로직 검증(Connectivity & Logic): 내가 작성한 @Controllerd의 URL이 제대로 동작하는지, 그리고 DB에 데이터가 정상적으로 저장되거나 수정되었는지 확인.
